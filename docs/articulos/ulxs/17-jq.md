# jq

> grep para JSON

**2012 · Stephen Dolan · Procesador de JSON en línea de comandos**

## ¿Por qué?

Las APIs devuelven JSON. Los corpus anotados se distribuyen en JSON. Las herramientas de PLN exportan resultados en JSON. Pero JSON no es texto plano en el sentido de `grep`: su estructura anidada hace que las búsquedas por línea no sirvan. `jq` es a JSON lo que `grep` y `sed` son al texto plano: una forma de filtrar, transformar y extraer datos sin escribir código.

## ¿Qué?

Procesador de JSON en línea de comandos con su propio lenguaje de filtros. Un programa `jq` es una cadena de transformaciones que recibe JSON y devuelve JSON (u otro tipo de valor). La expresión más simple, `.`, devuelve la entrada sin modificar, formateada.

## ¿Para qué?

Extracción de campos de respuestas de APIs, transformación de resultados de spaCy o de modelos de lenguaje, filtrado de corpus en JSON, conversión entre formatos. Disponible también como librería en Python, JavaScript y otros lenguajes.

### Sintaxis

| Construcción | Significado |
|---|---|
| `.` | identidad: pasa la entrada sin modificar |
| `.campo` | acceso a campo de objeto |
| `.[n]` | acceso a elemento de array |
| `\| ` | pipe: encadena filtros |
| `select(condición)` | filtra elementos que cumplen la condición |
| `[.[] \| filtro]` | mapea un filtro sobre todos los elementos |

### Ejemplo

```json
[
  {"token": "corriendo", "lema": "correr", "pos": "VERB"},
  {"token": "rápido",    "lema": "rápido", "pos": "ADJ"},
  {"token": "el",        "lema": "el",     "pos": "DET"}
]
```

```jq
.[] | .lema                          (todos los lemas)
[.[] | select(.pos == "VERB")]       (solo verbos)
[.[] | {(.pos): .lema}]              (objeto pos → lema)
```
